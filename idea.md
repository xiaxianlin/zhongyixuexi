# 中医经典学习软件 · 产品文档（本地 PC 版）

> **定位**：一款**本地运行的 PC 桌面专业学习软件**，把中医经典（本草、内经、方剂）做成「能读懂、能检索、能记住」的学习工具。
>
> **核心约束（本版基调）**：
> - **只做 PC**（Windows / macOS，可选 Linux），不做移动端。
> - **本地应用**：所有数据、进度、笔记、内容包均在本地；**无账号、无云同步、无服务端**。
> - **唯一联网点**：调用外部 AI API（LLM / 图像 / TTS），由用户自行配置 API Key。
> - **专业学习导向**：核心是「阅读 + 理解 + 记忆 + 检索 + 笔记」，不做社交、裂变、订阅墙。

---

## 1. 产品定位与目标用户

**一句话**：中医经典的「专业本地学习器」——古文权威底本 + 高质量白话解读 + AI 插图 + 间隔重复记忆闭环，全部离线可用。

**目标用户**：

| 优先级 | 人群 | 核心诉求 |
|---|---|---|
| P0 | 中医院校学生 / 备考者 | 背诵原文、理解功效、刷题记忆、高效检索 |
| P1 | 深度中医爱好者 / 临床进修者 | 系统学习、建立知识体系、做笔记沉淀 |
| P2 | 传统文化 / 医史研究者 | 多版本原文对照、引用、导出 |

**差异化**：多数中医 App/电子书要么是"词典 + 题库"重工具、要么是"扫描 PDF"无解读。我们的壁垒 = **权威原文 + 结构化解读 + 统一风格插图 + 间隔重复记忆** 的内容产品化能力，并因**完全本地**带来隐私、离线、零订阅负担的优势。

---

## 2. 整体模块架构

**模块化 + 本地分层**：

- **内容库模块**（核心）：内置内容包 + 可导入扩展包。
- **阅读模块**（主场景）：PC 大屏多栏布局。
- **学习闭环模块**：记忆卡（Anki 式间隔重复）、测验、复习计划。
- **检索 / 知识图谱模块**：全文检索、术语关联、药材-方剂-症状映射。
- **笔记模块**：Markdown 笔记，本地存储、导出。
- **工具模块**：AI 智能问答、AI 重绘插图、TTS 朗读（均走用户 API Key）。
- **设置模块**：AI API Key 配置、字体/主题、数据备份与导出。

> 说明：原移动版设想中的「用户/社区模块」「后台 CMS 服务端」**移除**。内容生产在**开发侧**完成（见 §7），打包进应用或作为内容包分发，软件本身不含服务端。

## 3. PC 阅读体验设计（核心场景，发挥大屏优势）

PC 大屏是本产品的**结构性优势**：移动端只能竖向单栏切换，PC 可以**多窗并排常驻**，适合长时间深度学习。

### 3.1 主阅读界面：多栏可布局工作台

默认三栏，用户可自由拖拽 / 折叠 / 弹出独立窗口：

| 栏位 | 内容 | 说明 |
|---|---|---|
| **左·原文栏** | 古文（可切繁简、带拼音/注音），仿宋衬线体，字号/行距可调 | 点击词 → 浮窗释义 + 关联条目跳转 |
| **中·解读栏** | 白话逐句译文 + 医理点拨 + 关键知识卡（君臣佐使 / 上品无毒等） | 与原文逐句对齐滚动（可锁定同步） |
| **右·资源栏** | AI 插图（可缩放/重绘）+ 经络图 / 药材实拍图 + 关联方剂/篇章 | 可折叠为笔记栏 |

### 3.2 PC 专属能力（移动端做不到的）

- **多版本对照**：同一篇章并排显示不同辑本 / 注本（如《神农本草经》孙星衍本 vs 顾观光本），逐句差异高亮。
- **原文 ↔ 译文锁定滚动**：左古文、中白话同步联动，定位精确到句。
- **知识图谱侧边栏**：当前药材 → 五味 / 归经 / 毒性 / 关联方剂 / 相须相使，可点击展开关系图。
- **独立笔记窗口**：分屏常驻，边读边记，支持 `[[双链]]` 引用任意条目（类 Obsidian）。
- **多 Tab / 多窗**：同时打开多个条目对照研读。
- **快捷键体系**：翻条、高亮、加记忆卡、跳转全键盘操作，适合桌面高效学习。

### 3.3 交互细节

- **检索式阅读**：选中"脾虚" → 全库高亮 + 结果列表侧栏。
- **AI 问答**：右侧常驻问答框，基于本地检索 + 用户 LLM API，回答标注引用条目可跳转。
- **TTS**：原文慢读 / 白话快读双轨，倍速、后台播放（PC 可边听边做笔记）。
- **AI 重绘插图**：编辑 Prompt → 调用用户图像 API 生成新图，本地保存。

## 4. 学习闭环模块（专业学习核心）

> 这是区别于"普通阅读器"的关键，面向**记忆与掌握**。

- **间隔重复记忆卡（SRS）**：Anki 式算法（SM-2）。卡片类型：
  - 原文 → 译文 / 解读
  - 药名 → 功效 / 归经 / 性味
  - 插图 → 名称
  - 篇章标题 → 核心医理
- **每日复习计划**：本地计算到期卡片，streak 本地统计（不联网）。
- **测验**：药性匹配、方剂组成、医理判断，记录正确率，错题自动转记忆卡。
- **学习仪表盘**：进度、掌握度热力图、薄弱条目推荐复习。

---

## 5. 检索与知识图谱

- **全文检索**：基于 SQLite FTS5，支持书名 / 章节 / 关键词（"人参""脾虚""补五脏"）。
- **结构化筛选**：按 五味 / 归经 / 毒性 / 上下品 / 症状 / 方剂 过滤。
- **知识图谱**：药材—方剂—症状—经络的多维关联，可视化关系网络，点击跳转。
- **术语词典**：点击任意术语 → 本地弹窗（定义 + 出处 + 关联条目 + 插图）。

## 6. 数据模型（本地 SQLite）

```json
// 单条目结构（同时持久化为 SQLite 行 + 本地文件）
{
  "book_id": "shennong_bencao",
  "title": "神农本草经",
  "volume": "上卷",
  "chapter": "上品",
  "entry_id": "renshen",
  "original": "人参，味甘微寒。主补五脏，安精神，定魂魄……",
  "modern": "人参味道甘、性微寒。主要作用是……",
  "explanation": "医理点拨：……",
  "properties": { "flavor": ["甘"], "nature": "微寒", "meridian": ["脾","肺","心"], "grade": "上品", "toxicity": "无毒" },
  "images": [{ "path": "local://images/renshen_01.png", "prompt": "…", "alt": "人参山谷图" }],
  "audio": { "original": "local://audio/renshen_orig.mp3", "modern": "local://audio/renshen_mod.mp3" },
  "keywords": ["补五脏", "上品", "人参"],
  "relations": { "formulas": ["四君子汤"], "symptoms": ["气虚","脾虚"] }
}
```

**本地存储设计**：
- `app.db`（SQLite）：条目结构化数据、FTS 全文索引、用户进度、笔记、记忆卡调度、测验记录。
- `assets/`：图片、音频（本地文件，DB 存相对路径）。
- `content-packs/`：可导入的内容扩展包（见 §7）。

## 7. 内容生产流程（开发侧，非服务端）

**没有 CMS 服务端**：内容在**开发期**用 AI + 专家审核的流水线生产，产出的成果作为「内容包」内置进安装包或作为 `.pack` 文件分发，用户**手动导入**（从官网 / 网盘下载后拖入应用）。软件运行时不连接任何内容服务器。

**单条目生产 Pipeline**：

```
[1 原文校勘] → [2 AI 生成初稿] → [3 专家审核] → [4 结构化标注] → [5 插图/音频] → [6 打包内容包]
```

1. **原文校勘**：以公版权威版本为底本（《神农本草经》孙星衍辑本 / 顾观光本互校）。**古文公版，但现代校注本有版权——必须用公版底本或自校。**
2. **AI 生成初稿**：原文 + 固定 System Prompt → 结构化 JSON（白话 / 解读 / 知识卡 / 插图 Prompt / 关键词 / 性味归经）。温度 0.3-0.5。
3. **专家审核**（中医执业医师 / 学院教师）：审功效准确性、是否触及处方 / 诊疗建议。**法律与质量双保险，不可省略。** 配《审核 checklist》《争议回退机制》。
4. **结构化标注**：五味、归经、毒性、上下品、关联方剂 / 症状。
5. **插图 / 音频**：批量生成（见 §8），本地化存储。
6. **打包**：导出为版本化的内容包（`.pack`），内置首发 + 后续作为扩展包发布。

**成本估算**：单条目 ≈ 文本 ¥0.5-2 + 插图 ¥0.5-1 + 音频 ¥0.3；专家审核 ¥5-20/条。本草 365 条冷启动约 ¥3k-8k。

## 8. AI 服务设计

**区分两个场景**：

### A. 开发期内容生产（开发者用，见 §7）
- 文本：Claude Sonnet 4.6 / 国内通义 / DeepSeek，固定 Prompt + few-shot 黄金样本（人参条），结构化输出。
- 插图：即梦 / 可灵（国内）或 Midjourney（海外）。一致性 = 固定风格参考图 + 固定 Seed + 统一模板（"宋代医书风 + 水墨淡彩 + 留白"）。
- TTS：古文需**拼音标注词典**覆盖多音字 / 通假字 / 异读（"参"在人参读 shēn），SSML 标注后合成。

### B. 运行期用户功能（走用户自配 API Key，软件本身不内置付费）
- **AI 智能问答**：本地检索（SQLite FTS5 或本地向量）top-k 条目 → 拼入 Prompt → 调用用户 LLM → 输出"引用原文 + 解读"，答案标注来源条目可跳转。
- **AI 重绘插图**：编辑 Prompt → 调用用户图像 API → 本地保存。
- **个性化解读 / TTS**：按需调用用户 API 生成。
- **关键红线**：System Prompt 显式禁止任何"某病吃什么药 / 剂量"的诊疗建议，用户端问答同样拦截。

**API Key 管理**：设置面板让用户填入 Key（OpenAI / Anthropic / 国内厂商），Key 加密存本地，**不离开本机**（除发往用户指定的 API 端点）。

## 9. 技术架构（本地优先）

```
┌──────────────────────────────────────────────────┐
│  桌面壳（Electron 或 Tauri）                       │
│  ┌────────────────────────────────────────────┐  │
│  │ 前端 UI（React + TypeScript）               │  │
│  │ 阅读 / 记忆卡 / 检索 / 图谱 / 笔记 / 设置    │  │
│  └───────────────┬────────────────────────────┘  │
│                  │ 本地 IPC                       │
│  ┌───────────────▼────────────────────────────┐  │
│  │ 本地运行时                                    │  │
│  │ - SQLite（app.db + FTS5）                    │  │
│  │ - 本地文件系统（assets / content-packs）      │  │
│  │ - AI API 客户端（用户 Key，仅此处联网）       │  │
│  └─────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
        │ 仅此处出网：用户配置的 AI API（LLM / 图像 / TTS）
        ▼
   外部 AI 服务（用户自带 Key）
```

**技术选型**：
- **桌面壳**：推荐 **Electron + React**（生态成熟、better-sqlite3 集成简单、国内打包 Win/Mac 顺、踩坑少）；若追求体积/内存（包 ~10MB vs Electron ~100MB）可选 **Tauri**，但需 Rust、部分 Node 库不可用。
- **存储**：SQLite（better-sqlite3 同步、性能好）+ FTS5 全文检索。
- **向量检索**（可选，智能问答用）：数据量不大（几百~千条），可用 sqlite-vec / 纯内存检索，无需独立向量库。
- **放弃 Flutter Desktop 的理由**：转为 PC 富内容（富文本、Markdown、Anki、图谱可视化）后，Web 技术栈生态更成熟、组件更丰富。

**离线 / 更新策略**（严守"除 AI API 外不联网"）：
- 首版内容**内置**于安装包。
- 内容更新通过**手动导入内容包文件**（`.pack`）——用户从官网/网盘下载后拖入应用，应用本身不主动联网拉取。
- 软件更新走常规桌面更新机制（可选）。
- 所有进度、笔记、记忆卡 100% 本地；提供**本地备份 / 导出 / 导入**（整个 `app.db` + assets 打包），换机可迁移。

## 10. 合规与风险

| 风险 | 说明 | 应对 |
|---|---|---|
| **医疗内容合规** | 功效描述可能被认定涉及"疾病预防/治疗" | 严格定位为**典籍阅读 + 知识科普**；不给剂量、不推处方、不做诊疗；专家审核闭环；显著免责声明 |
| **药材安全** | 有毒 / 管制药材（附子、马钱子等） | 条目加红色警示卡，禁任何使用建议 |
| **版权** | 古文公版，但现代校注 / 译本有版权 | 公版底本自校；白话由 AI+专家原创；插图 AI 生成排查训练数据风险 |
| **AI 生成内容标识** | 监管要求标识 | 插图 / AI 解读标注"AI 辅助生成" |
| **用户 API Key 安全** | Key 是敏感凭证 | 本地加密存储，不明文日志，仅发往用户指定端点 |

> 单机软件分发仍需：用户协议、隐私政策、医疗免责声明；按发行渠道评估备案要求。

## 11. 商业模式

单机本地软件，去掉订阅墙 / B 端授权，建议：

- **买断制**（主力）：按版本买断（如 ¥99-199），含全库内容 + 全功能 + 后续小版本免费更新；大版本升级付费。
- **内容扩展包**（可选）：新书 / 新专题作为付费 `.pack` 内容包单独售卖。
- **免费试用版**（引流）：内置本草上品部分条目 + 基础阅读 + 记忆卡，付费解锁全库 / 图谱 / 测验。

## 12. UI/UX 设计规范

- **颜色**：主色 #5C4033（古墨）、辅 #EDE4D5（米纸）、强调 #A67C5D；深色模式适配长时间学习。
- **排版**：原文 18-22px 衬线，白话 16-20px 无衬线；行高 1.6-1.8，段距 12-16px。
- **插图风格**：统一"宋代医书风 + 水墨淡彩"，≥1024px，Seed + 风格参考图控一致性。
- **布局**：桌面优先，支持分屏、多窗、快捷键；响应常见 PC 分辨率（1280 起，适配 4K 缩放）。
- **可访问性**：动态字体、高对比、TTS 朗读。
- **动画**：克制（页面淡入、插图渐显），不干扰阅读。

## 14. 关键指标（本地可采集，匿名，不出机）

- **参与度**：单条完成率 >70%、日均学习时长 >20min、记忆卡每日复习完成率。
- **学习效果**：记忆卡掌握度分布、测验正确率趋势、错题收敛。
- **留存**（本地）：7/30 日活跃打开次数。
- **内容健康**：AI 重绘触发率（插图吸引力）、问答使用率。

## 15. 待决策的开放问题

1. **桌面壳选型**：Electron（成熟、上手快、包大）vs Tauri（轻量、需 Rust）？建议 **Electron** 起步降风险。
2. **内容分发边界**：严守"不联网"则只能手动导入 `.pack`；是否接受一个**极轻量的版本检查 + 静态文件下载**（仅查新内容包、不传任何用户数据）？需确认"不联网"的严格程度。
3. **AI 模型默认支持**：默认对接哪些厂商（OpenAI / Anthropic / 通义 / DeepSeek / 即梦）？影响首版 Key 配置文档。
4. **内容深度**：MVP 先做"本草 + 内经"两本，还是聚焦一本做透？建议聚焦本草上品做深，再横向扩。
