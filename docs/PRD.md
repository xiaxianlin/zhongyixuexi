## 1. 概述

### 1.1 产品定位

一款**本地运行的 PC 桌面专业学习软件**，用户自行导入书籍（首期 EPUB），软件将其解析为可阅读、可检索、可记忆的章节与段落结构，并提供 AI 白话解读、问答与间隔重复记忆等学习增强。

软件**尤其适合中医经典与古籍**（古文逐句解读、记忆、配图），但内容来源完全由用户决定，软件本身不内置、不分发任何书籍。

### 1.2 核心约束

| 编号 | 约束 | 说明 |
|---|---|---|
| C-1 | 只做 PC | Windows / macOS，不做移动端、不做 Web 在线版 |
| C-2 | 本地优先 | 所有数据、解析结果、笔记、记忆卡均在本地；无账号、无云同步、无服务端 |
| C-3 | 联网最小化 | 唯一联网点是调用外部 AI API；**默认 DeepSeek**，用户可自行配置 Key 与厂商 |
| C-4 | 专业学习导向 | 核心 = 导入 + 阅读 + 理解 + 记忆 + 检索 + 笔记，不做社交、裂变、推荐流 |
| C-5 | 内容零内置 | 不预置任何书籍；所有内容由用户自行导入，软件仅做解析与学习增强 |
| C-6 | 首期仅 EPUB | 第一期只支持 EPUB 导入解析；PDF / OCR 纳入后续阶段 |

### 1.3 价值主张

- **任意导入**：EPUB 即拖即用，把已有的中医典籍、教材变成结构化学习材料。
- **智能解析 + 段级校对**：自动切分章节/段落，并提供段级编辑工具修正解析误差。
- **AI 增强**：逐句白话、医理点拨、问答（中医典籍场景尤其有效，默认 DeepSeek）。
- **记忆闭环**：间隔重复 + 测验，服务"记住"而非"读过"。
- **可控**：完全本地、离线可用（AI 功能除外）、隐私不出本机。

## 2. 目标用户与场景

### 2.1 用户分层

| 优先级 | 人群 | 核心诉求 |
|---|---|---|
| P0 | 中医院校学生 / 备考者 | 导入教材/典籍，背诵原文、理解功效、刷题记忆 |
| P1 | 深度爱好者 / 临床进修者 | 系统学习导入的典籍、做笔记沉淀 |
| P2 | 传统文化 / 医史研究者 | 多版本对照、引用、导出 |

### 2.2 典型场景

- **S1 导入学习**：拖入《神农本草经》EPUB → 自动解析上中下三品与各药条目 → 段级微调校对 → 三栏阅读 → 加入记忆卡。
- **S2 段级校对**：解析后某段断句错误 / 混入页眉 → 选中该段编辑修正 / 合并拆分 → 保存，不影响已生成的 AI 解读（按段 ID 稳定映射）。
- **S3 备考记忆**：今日到期复习卡片 → 翻卡自测 → 错题转卡。
- **S4 主题检索**：搜"脾虚" → 全库（跨多本导入书）命中段落 → 笔记 `[[双链]]` 引用。
- **S5 AI 辅助**：选中段落 → DeepSeek 生成白话解读 → 右侧问答框提问 → 基于本书检索回答 + 标注来源。

## 3. 功能需求

> 优先级：P0 = MVP 必须；P1 = 紧随其后；P2 = 后续迭代。每个功能含**验收标准（AC）**。

### 3.1 模块总览

| 模块 | 编号前缀 | 说明 |
|---|---|---|
| 导入与解析 | IMP | EPUB 导入、章节/段落解析、段级校对（核心） |
| 书库管理 | LIB | 多本书的浏览、元信息、进度 |
| 阅读 | RD | PC 大屏多栏阅读 |
| 学习闭环 | LRN | 记忆卡 / 测验 / 复习 |
| 检索与知识图谱 | SRH | 全文检索 / 筛选 / 关系网络 |
| 笔记 | NOTE | Markdown + 双链 + 导出 |
| AI 工具 | AI | 解读 / 配图 / 标注 / 卡片生成 / 问答 / TTS（走用户 Key，默认 DeepSeek） |
| 设置与数据 | SET | API Key / 主题 / 备份导入导出 |

---

### 3.2 导入与解析模块（IMP）—— 核心

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| IMP-01 | EPUB 导入 | P0 | 拖入或选择 `.epub`；解包并解析为 书籍 + 章节树（基于 OPF spine / NCX / Nav TOC）+ 段落 |
| IMP-02 | 段落解析 | P0 | 章节正文按语义切分为段落（保留段 ID）；保留原始 HTML 结构信息（标题层级、列表等）以辅助识别 |
| IMP-03 | 段级编辑 | P0 | 可视化校对界面：**段落级**增删改、合并/拆分段落、调整段落顺序、批量清理噪声（页眉/页脚/水印/多余空行）；原文与解析预览对照 |
| IMP-04 | 章节编辑 | P0 | 章节级合并/拆分/重排、编辑章节标题、修正层级（卷-品-篇） |
| IMP-05 | 原文清理 | P1 | 断行拼接、空格归一、编码修复；古文可选繁简转换、自动注音标注 |
| IMP-06 | 解析质量提示 | P0 | 解析失败/存疑的章节与段落明确标记，引导校对 |
| IMP-07 | 重新解析 | P1 | 修改解析参数后可重新解析；**段落采用稳定 ID**，已校对内容与已生成 AI 解读按段 ID 映射保留 |
| IMP-08 | 导入去重 | P1 | 同名/同内容书提示是否覆盖或作为新副本 |

> **PDF / OCR**：第一期不支持。计划在后续阶段加入 PDF（带大纲优先）与扫描型 PDF 的 OCR。首期专注把 EPUB 链路做扎实。

> 所有 AI 功能走用户自配 API Key（见 SET-01），**默认厂商 DeepSeek**。导入与解析本身不依赖 AI；AI 用于对已解析内容做增强。

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| AI-01 | 白话解读生成 | P0 | 选中段落/章节 → DeepSeek 生成逐句白话 + 医理点拨；本地缓存，可重新生成；与原文锁定对齐 |

## 4. 非功能需求

### 4.1 性能

| 编号 | 指标 | 目标 |
|---|---|---|
| NFR-P1 | 冷启动 | ≤ 3 秒 |
| NFR-P2 | 章节/段落打开 | ≤ 200ms（本地命中） |
| NFR-P3 | 全文检索 | ≤ 300ms（万段落级） |
| NFR-P4 | EPUB 解析 | 中型书（数百章）≤ 5 秒 |
| NFR-P5 | 内存占用 | 稳态 ≤ 400MB |

### 4.2 可靠性与数据安全

- 本地 SQLite 写入事务保护，异常崩溃不丢数据。
- 笔记与进度自动保存。
- API Key 本地加密（OS keychain / 加密存储），不明文落盘、不进日志。
- 备份文件完整性校验。
- **段落与章节采用稳定 ID**，重新解析/段级编辑/迁移不破坏笔记、记忆卡、AI 解读的引用关系。

### 4.3 兼容性

- Windows 10/11（x64）、macOS 12+（Intel + Apple Silicon）。
- 适配常见分辨率（1280 起）与高 DPI / 4K 缩放。
- 离线场景：除 AI 功能外全部可用；导入/解析/阅读/记忆/检索全离线。

### 4.4 可访问性

- 动态字体大小、高对比模式、TTS 朗读、完整键盘操作。

## 5. 数据模型

### 5.1 核心实体

```json
// 书籍
{
  "book_id": "uuid",
  "title": "神农本草经",
  "author": "...",
  "source_format": "epub",
  "source_file": "local://files/shennong.epub",
  "cover": "local://covers/xxx.png",
  "category": "本草",
  "imported_at": 0,
  "parse_version": 2
}

// 章节（层级容器，存标题与树结构；正文下放到段落）
{
  "chapter_id": "uuid",
  "book_id": "uuid",
  "parent_id": "uuid | null",   // 层级：卷-品-篇
  "order_index": 5,
  "level": "品",
  "title": "上品·人参"
}

// 段落（正文最小编辑/引用单元，稳定 ID）
{
  "paragraph_id": "uuid",
  "chapter_id": "uuid",
  "order_index": 3,
  "text": "人参，味甘微寒……",
  "content_modern": "...",        // AI 生成（DeepSeek），可空
  "content_explanation": "...",   // AI 生成，可空
  "edited": true,                 // 用户是否手动编辑过
  "parse_hash": "..."             // 用于重新解析时映射保留
}
```

> 笔记、记忆卡、书签、阅读进度、AI 解读均通过 `paragraph_id` / `chapter_id` 精确绑定，支持段级粒度。

### 5.2 本地存储结构

| 存储 | 内容 |
|---|---|
| `app.db`（SQLite） | books / chapters / paragraphs、FTS5 全文索引、阅读进度、笔记、记忆卡调度、测验记录、AI 缓存、设置 |
| `assets/` | AI 生成的图片、音频 |
| `files/` | 用户导入的原始 EPUB 文件 |

### 5.3 核心表（概要）

- `books`：书籍元信息与来源文件。
- `chapters`：章节层级（稳定 ID，支持重排）。
- `paragraphs`：段落正文（稳定 ID，段级编辑单元）。
- `reading_progress`：阅读进度（段级）。
- `notes`：笔记（含双链解析，可绑段）。
- `cards` / `review_log`：记忆卡与复习记录（SM-2 调度）。
- `quiz_results`：测验记录与错题。
- `ai_cache`：AI 生成内容缓存（关联 paragraph_id，可失效重生成）。
- `settings`：应用设置（含加密 API Key）。
- `fts_paragraphs`：FTS5 虚拟表（段落全文索引）。

## 6. 导入解析流程（运行期）

> 无内容服务端、无开发期生产。用户导入后由本地解析 + 可选 AI 增强完成。

```
[1 导入 EPUB] → [2 解包解析] → [3 章节/段落切分] → [4 段级校对] → [5 入库阅读] → [6 AI 增强(按需)]
```

1. **导入文件**：EPUB 拖入或选择。
2. **解包解析**：ZIP 解包，解析 OPF spine + NCX/Nav TOC → 章节树与 XHTML 正文。
3. **章节/段落切分**：生成层级化章节 + 段落，建立稳定 ID。
4. **段级校对**：用户在 IMP-03 界面段级编辑、合并/拆分、清理噪声。
5. **入库阅读**：进入书库与阅读流程。
6. **AI 增强（按需，默认 DeepSeek）**：白话解读、配图、标注、记忆卡、问答。

> 原专家审核流程不适用（无法对用户任意导入内容审核）；改为：显著免责声明 + AI 生成内容标识 + 问答红线拦截。

## 7. AI 服务设计

### 7.1 运行期增强（默认 DeepSeek，走用户 Key）

- **白话解读**：原文 + 固定 Prompt（中医典籍含医理点拨模板）→ 逐句结构化输出。温度 0.3-0.5。
- **问答 / RAG**：本地检索（FTS5 或本地向量）top-k 段落 → 拼入 Prompt → DeepSeek → 输出带来源引用。
- **标注 / 记忆卡**：抽取关键词、要点，生成结构化数据与卡片草稿。
- **配图 / TTS**：DeepSeek 暂不支持；需用户另配图像/TTS 厂商（P2 功能）。
- **红线**：System Prompt 禁止任何诊疗/剂量建议，问答端关键词兜底拦截。

### 7.2 缓存与成本

- AI 生成内容本地缓存（`ai_cache`，关联 paragraph_id），避免重复调用与重复计费。
- 调用前预估/提示 token 消耗，长内容分段生成。
- 失败降级：见 AI-07。

## 8. 技术架构

```
┌──────────────────────────────────────────────────┐
│  Electron 桌面壳（Node 主进程 + Chromium 渲染）    │
│  ┌────────────────────────────────────────────┐  │
│  │ 渲染进程 UI（React + TypeScript + Vite）    │  │
│  │ 导入校对 / 阅读 / 记忆卡 / 检索 / 笔记 / 设置│  │
│  └───────────────┬────────────────────────────┘  │
│                  │ IPC（contextBridge + invoke）   │
│  ┌───────────────▼────────────────────────────┐  │
│  │ 主进程（Node.js）                            │  │
│  │ - EPUB 解析（node-stream-zip + xml parser） │  │
│  │ - SQLite（better-sqlite3）+ FTS5            │  │
│  │ - 文件系统（assets / files）                  │  │
│  │ - AI API 客户端（fetch，仅此处联网）        │  │
│  └─────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
        │ 仅此处出网：DeepSeek（默认）/ 用户配置的 AI API
        ▼
   外部 AI 服务（用户自带 Key）
```

### 8.1 技术选型

- **桌面壳**：**Electron**（主进程 Node.js + Chromium；生态成熟、better-sqlite3/EPUB 库即取即用、打包顺）。代价是包体较大（~100MB+）、内存较高，学习类应用可接受。
- **前端**：**React + TypeScript + Vite**（HMR 开发体验好）。
- **IPC**：`contextBridge` + `ipcMain.handle` / `ipcRenderer.invoke`，`contextIsolation: true`、`nodeIntegration: false`。
- **EPUB 解析**：主进程 `node-stream-zip` 解包 + `fast-xml-parser` 解析 OPF/NCX/spine，段落切分后写 SQLite。
- **存储**：**SQLite（`better-sqlite3`）** + FTS5；主进程同步驱动，渲染进程不裸写 SQL。
- **HTTP / AI**：`undici` / 全局 `fetch` 调用 DeepSeek / 其他厂商。
- **向量检索（可选）**：`sqlite-vec`，千段落级无需独立向量库。
- **Key 安全存储**：Electron **`safeStorage`**（原生加密）或 `keytar`。
- **打包 / 更新**：**electron-builder** + **electron-updater**。

### 8.2 更新策略（前端热更新 + 整包更新）

- **前端（React UI）**：Electron 中前端资源本就是外部文件，天然支持热更——下载新版 bundle → 替换 → `win.reload()`，进程不退出，用户数据保留。绝大多数 UI/交互迭代无感更新。
- **主进程 / 原生模块**：用 **electron-updater** 差量更新，低频重启。涉及主进程逻辑、原生依赖、Electron 版本时才触发。
- DB schema 变更走版本化迁移，不可破坏已有稳定 ID。

### 8.3 离线与数据

- 全部解析结果、进度、笔记、记忆卡 100% 本地。
- 本地备份/导出/导入支持换机迁移。
- 联网仅 AI API（默认 DeepSeek，用户自配 Key）。

## 9. 合规与风险

| 风险 | 说明 | 应对 |
|---|---|---|
| **内容版权** | 用户导入的书籍可能涉及版权 | 软件不分发任何内容；用户协议明确：内容由用户自行导入并自负授权责任；软件仅为本地解析与学习工具 |
| 医疗内容合规 | 中医典籍的功效描述 | 定位为阅读/学习工具；不给剂量、不推处方、不做诊疗；问答红线拦截；显著免责声明 |
| 内容准确性 | 用户导入内容 + AI 生成未经专家审核 | AI 生成内容标注"AI 辅助生成"；免责声明提示用户自行判断 |

| EPUB 解析误差 | 不同 EPUB 制作质量差异 | IMP-03/04 段级+章级校对工具 + 质量提示；不保证 100% 准确 |

法务前置：用户协议、隐私政策、医疗免责声明、版权免责条款；按发行渠道评估备案要求。

## 13. 待决策开放问题

1. **EPUB 解析实现**：主进程 Node 库（`node-stream-zip` + `fast-xml-parser`）vs 渲染进程 JS 库。建议主进程 Node，与 better-sqlite3 写入一致、避免大对象过 IPC。
2. **段落切分粒度**：按 HTML 块级元素 / 按句 / 按自然段？影响段级编辑与解读对齐粒度。建议自然段为主，长段可选按句细分。
3. **向量检索首期是否引入**：FTS5 关键词检索已够问答 RAG 用？还是首期就上 `sqlite-vec` 语义检索？建议 FTS5 起步，语义检索按需。
4. **配图 / TTS 厂商**：DeepSeek 不支持，首期是否先不做（P2 推迟），还是默认对接某家（如即梦图像）？
