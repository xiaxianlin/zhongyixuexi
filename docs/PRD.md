> 1. **内容来源改为内置**：随包内置三本中医经典（《素问》《灵枢》《难经》），不再支持用户导入 EPUB。导入解析工具链保留为离线内容生产用途，不在应用运行路径内。

## 1. 概述

### 1.1 产品定位

一款**本地运行的 PC 桌面中医经典学习软件**，随包内置《黄帝内经·素问》《黄帝内经·灵枢》《黄帝八十一难经》三部经典，软件将其组织为可阅读、可检索、可记忆的章节与段落结构，并提供 AI 段级白话解读与医理点拨、全文检索、段绑定笔记与阅读足迹统计。

软件**本地优先**：所有数据、AI 解读缓存、笔记、阅读进度均在本地；无账号、无云同步、无服务端；唯一联网点是用户自配 Key 的 AI API（默认 DeepSeek）。

### 1.2 核心约束

| 编号 | 约束 | 说明 |
|---|---|---|
| C-1 | 只做 PC | Windows / macOS，不做移动端、不做 Web 在线版 |
| C-2 | 本地优先 | 所有数据、解读缓存、笔记、进度均在本地；无账号、无云同步、无服务端 |
| C-3 | 联网最小化 | 唯一联网点是调用外部 AI API；**默认 DeepSeek**，用户可自行配置 Key 与厂商 |
| C-4 | 专业学习导向 | 核心 = 阅读 + 理解 + 检索 + 笔记 + 足迹，不做社交、裂变、推荐流 |
| C-5 | 内置经典 | 随包预置《素问》《灵枢》《难经》三部；不再支持用户导入（v3.0） |
| C-6 | AI 走用户 Key | 所有 AI 功能依赖用户自配 API Key；未配置时降级为纯本地阅读 |

### 1.3 价值主张

- **开箱即读**：内置三部中医核心经典，无需导入即可开始系统学习。
- **AI 段级解读**：逐段生成白话译文、医理点拨与综合解读，中医典籍场景尤其有效（默认 DeepSeek）。
- **全文检索**：FTS5 跨书检索，命中段落可一键定位。
- **段绑定笔记**：阅读时围绕具体段落沉淀心得。
- **阅读足迹**：仪表盘呈现阅读进度、解读覆盖率、笔记量与热力图。
- **可控**：完全本地、离线可用（AI 功能除外）、隐私不出本机。

## 2. 目标用户与场景

### 2.1 用户分层

| 优先级 | 人群 | 核心诉求 |
|---|---|---|
| P0 | 中医院校学生 / 备考者 | 逐段读懂《内经》《难经》原文，理解医理，做笔记沉淀 |
| P1 | 深度爱好者 / 临床进修者 | 系统学习经典，检索主题，对照理解 |
| P2 | 传统文化 / 医史研究者 | 全文检索、引用、定位 |

### 2.2 典型场景

- **S1 逐段精读**：打开《素问》→ 选章 → 选段 → 一键 AI 生成白话/医理/解读 → 三栏对照理解。
- **S2 主题检索**：搜"脾虚" → 跨三书命中段落 → 点击跳转到书籍详情对应段。
- **S3 段落笔记**：读到关键段 → 添加笔记 → 抽屉查看该段所有笔记。
- **S4 足迹回看**：仪表盘查看已读章节数、AI 解读覆盖率、阅读热力图。
- **S5 AI 辅助**：选段 → DeepSeek 生成白话 → 解读面板展示 → 可重新生成（覆盖）。

## 4. 非功能需求

### 4.1 性能

| 编号 | 指标 | 目标 |
|---|---|---|
| NFR-P1 | 冷启动 | ≤ 3 秒 |
| NFR-P2 | 章节/段落打开 | ≤ 200ms（本地命中） |
| NFR-P3 | 全文检索 | ≤ 300ms（千段落级） |
| NFR-P4 | AI 段级解读 | 单段 ≤ 15 秒（含网络） |
| NFR-P5 | 内存占用 | 稳态 ≤ 300MB |

### 4.2 可靠性与数据安全

- 本地 SQLite 写入事务保护，异常崩溃不丢数据。
- 阅读进度与笔记自动保存。
- API Key 本地加密（safeStorage + AES fallback），不明文落盘、不进日志。
- **段落与章节采用稳定 ID**，AI 解读/笔记/进度按段 ID 绑定，不因内容调整失效。

### 4.3 兼容性

- Windows 10/11（x64）、macOS 12+（Intel + Apple Silicon）。
- 适配常见分辨率（1280 起）与高 DPI / 4K 缩放。
- 离线场景：除 AI 解读外全部可用；阅读/检索/笔记全离线。

### 4.4 可访问性

- 动态字体大小、完整键盘操作。

## 5. 数据模型（当前实现）

### 5.1 内置数据格式

随包 `data/{nanjing,suwen,lingshu}-original.json`，结构见 `docs/dev/book-import-json.md`（作为内容生产中间格式）。每文件含 `book`（id/title/author/category）+ `chapters[]`（含 `paragraphs[]`），启动时由 `seedBuiltinContent()` 事务写入。

### 5.2 核心表（当前 schema，详见 `electron/db/schema.ts`）

| 表 | 说明 |
|---|---|
| `books` | 书籍元信息（内置经典） |
| `chapters` | 章节层级（parent_id 自引用，稳定 ID） |
| `paragraphs` | 段落正文（稳定 TEXT PK + 隐式 rowid + parse_hash + 软删） |
| `fts_paragraphs` | FTS5 虚拟表（content='paragraphs', trigram, ai/ad/au 触发器） |
| `reading_progress` | 段级阅读进度（按 book_id 唯一） |
| `notes` | 段绑定 Markdown 笔记（paragraph_id SET NULL） |
| `ai_cache` | AI 解读缓存（scope/kind/prompt_hash 命中） |
| `paragraph_analyses` | 段落解读版本化（active 唯一索引，modern/explanation/analysis） |
| `settings` | KV 设置 |
| `api_credentials` | AI 凭证（safeStorage 加密） |

> **schema 机制**：当前为单一 `CURRENT_SCHEMA`（version=2）+ 版本不符删库重建（仅开发期）。**生产打包前必须改为 forward-only 版本化迁移**（见 PROGRESS S8.1）。

### 5.3 本地存储结构

| 存储 | 内容 |
|---|---|
| `app.db`（SQLite，userData 目录） | 上述全部表 |
| `data/*.json`（随包） | 内置经典源数据 |

## 6. AI 服务设计（收敛形态）

### 6.1 段级解读（AI-01，默认 DeepSeek）

- 输入：段落原文 + 固定 Prompt（中医典籍含医理点拨模板）→ 结构化输出（白话/医理/解读）。
- 温度 0.3，JSON 模式。
- 输出写入 `paragraph_analyses`（active 版本）+ `ai_cache`。

### 6.2 缓存与成本

- `ai_cache` 按 `prompt_hash`（归一化 prompt + model + temp 的 sha256）命中，避免重复计费。
- 段落文本变化后 hash 变化，避免误命中。

### 6.3 红线与降级

- System Prompt 显式禁止诊疗/剂量建议。
- 输出端关键词兜底净化（中医剂量正则）。
- Key 未配置或调用失败：解读面板提示，不阻断阅读（AI-02）。

## 7. 技术架构

```
┌──────────────────────────────────────────────────┐
│  Electron 桌面壳（Node 主进程 + Chromium 渲染）    │
│  ┌────────────────────────────────────────────┐  │
│  │ 渲染进程 UI（React + TypeScript + Vite）    │  │
│  │ 书库/详情/检索/笔记/设置/足迹                │  │
│  └───────────────┬────────────────────────────┘  │
│                  │ IPC（{invoke,on} 信封）         │
│  ┌───────────────▼────────────────────────────┐  │
│  │ 主进程（Node.js）                            │  │
│  │ - 内置经典 seed（启动幂等）                  │  │
│  │ - SQLite（better-sqlite3）+ FTS5            │  │
│  │ - AI API 客户端（fetch，仅此处联网）        │  │
│  └─────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
        │ 仅此处出网：DeepSeek（默认）/ 用户配置的 AI API
        ▼
   外部 AI 服务（用户自带 Key）
```

### 7.1 技术选型

- **桌面壳**：Electron（主进程 Node.js + Chromium）。主进程强制 CJS（无 `type:module`），渲染进程 ESM（Vite）。
- **前端**：React 18 + TypeScript + Vite；状态管理 Zustand（仅会话/UI 缓存，持久化一律 SQLite）。
- **IPC**：`contextBridge` 暴露 `{invoke, on}`；`ipcMain.handle` 经 `handle()` 包成 `{__ok}` 信封；渲染端 `invokeRaw` 拆信封抛 `IpcError`。`contextIsolation: true`、`nodeIntegration: false`。
- **存储**：SQLite（`better-sqlite3`）+ FTS5（trigram）；主进程同步驱动，渲染进程不裸写 SQL。
- **HTTP / AI**：全局 `fetch` 调用 DeepSeek / 其他厂商。
- **Key 安全存储**：Electron `safeStorage` + 机器绑定 AES fallback。
- **打包 / 更新**：electron-builder（Phase 8，todo）。

### 7.2 更新策略（规划）

- **前端热更新**：下载新版 bundle → 替换 → `win.reload()`，进程不退出。
- **主进程 / 整包**：electron-updater 差量更新（Phase 8）。
- DB schema 变更：当前为删库重建（开发期），生产前需改为版本化迁移。

### 7.3 离线与数据

- 全部阅读内容、进度、笔记、解读缓存 100% 本地。
- 联网仅 AI API（默认 DeepSeek，用户自配 Key）。

## 8. 合规与风险

| 风险 | 说明 | 应对 |
|---|---|---|
| **内容版权** | 内置《素问》《灵枢》《难经》为 Public Domain 古籍 | 确认源文本公有领域地位；注明出处与所用底本 |

| 内容准确性 | 古籍文本 + AI 生成未经专家审核 | AI 生成内容标注；免责声明提示用户自行判断 |
| API Key 安全 | Key 为敏感凭证 | 本地加密存储，不明文日志，仅发往用户指定端点 |
