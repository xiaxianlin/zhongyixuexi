## 1. 背景与目标

### 1.1 当前形态（v3.0 详情页）

详情页是**章 / 段 / 析**三栏：

- **章（左）**：当前书所有章节的**平铺**列表，支持「重命名 / 新增 / 删除」。
- **段（中）**：选中章节后展示该章的**段落列表**；点选某段后右栏只展示该段的解读；支持「新增 / 编辑 / 合并 / 拆分 / 删除」段落。
- **析（右）**：展示**当前选中段落**的 AI 解读（解读 / 医理 / 白话），右上有「分析」按钮。

AI 解读为**段级**（`paragraph_analyses`，`kind='modern'`），一次只能解读一段，不存在对话能力。

### 1.2 痛点

1. **章平铺**：古籍的「篇 → 节」结构无法表达，长书目录难导航。
2. **段为最小单元**：阅读被切成一屏一段，破坏原文连贯性；AI 也只能逐段生成，成本高、上下文碎。
3. **析只读一段**：不能就当前章节追问、不能把原文片段喂给 AI、没有笔记 / 摘录的统一入口。
4. **无摘录概念**：看到关键句只能复制粘贴到外部，无法沉淀回库。
5. **古籍 / 现代书不分**：现代书也走「白话翻译」，逻辑不通。

### 1.3 本次目标（v3.1）

把详情页从「段级精读工作台」改造为「**章级阅读 + 多功能侧栏**」：

1. **书籍分类**：古籍 / 现代书，影响 AI 是否生成白话。
2. **章为树**：章节支持多级「篇 → 章 → 节」。
3. **段升章**：阅读区直接展示整章正文，不再强制分段；正文可被光标**选区**。
4. **选区三连**：选中文字弹出「摘录 / 写笔记 / 引用」。
5. **析改 Tab**：右侧改为 6 个 Tab —— **对话 / 解读 / 医理 / 白话 / 笔记 / 摘录**，默认进「对话」；Tab 头竖排。
6. **章级 AI**：AI 分析以**章**为单位，输出解读 + 医理（+ 白话，仅古籍）。
7. **可编辑**：阅读区右上的「AI 分析」按钮 + 章节正文可编辑。

## 3. 用户与场景

### 3.1 受影响场景

| 场景 | v3.0 | v3.1 |
|---|---|---|
| 精读一章 | 逐段点击，每段单独 AI | 打开章 → 整章连读 → 一键章级 AI |
| 抓重点 | 复制段落到外部 | 选中句子 → 摘录 / 笔记 / 引用到对话 |
| 多级目录 | 平铺，长书难找 | 树形目录，可折叠 |
| 追问 AI | 不支持 | 选中 → 引用 → 对话 Tab 追问 |
| 现代教材 | 也生成白话 | 现代书不生成白话，只解读 / 医理 |

### 3.2 典型用例（本次新增）

- **U1 章级精读**：进入《素问》「上古天真论」→ 右栏切到「解读」→ 点右上「AI 分析」→ 整章一次性出解读 + 医理 + 白话。
- **U2 摘录沉淀**：读到「恬淡虚无，真气从之」→ 选中 → 「摘录」→ 进「摘录」Tab 查看。
- **U3 引用追问**：选中一段脉象描述 → 「引用」→ 切到「对话」Tab，原文已自动作为上下文 → 输入「这里说的脉象和《难经》十八难是一回事吗？」。
- **U4 章节管理**：在某篇下「新增小节」→ 拖动或编辑标题；删除小节时其正文与摘录按 cascade / SET NULL 规则处理。
- **U5 编辑原文**：导入或 OCR 文本有错 → 阅读区「编辑」→ 修正后保存；相关摘录 / 笔记按重新锚定策略提示。

## 5. 选区与锚定（核心机制）

摘录、笔记引用、AI 引用都依赖**字内选区**，定义统一：

### 5.1 选区坐标

- **基准**：以 `chapters.content` 的纯文本为坐标空间。
- **字段**：`start_offset`、`end_offset`（UTF-16 code unit 偏移，与 JavaScript `String.prototype.slice` 一致）。
- **存证**：同时存 `excerpt_text`（被选中的原文），便于编辑后比对与展示。

### 5.2 重新锚定（content 变更后）

正文保存（RD-06）触发：

1. 用 `diff`（建议 Myers，见技术文档）计算旧 → 新的字符映射。
2. 仍可定位的选区：更新 offset。
3. 无法定位（被删 / 大幅改写）：`stale=1`，UI 在卡片上提示「原文已修改」，保留原文快照。

### 5.3 跨段落 / 跨节点

- 选区限定在**同一章**内（不允许跨章选）。
- 章节树移动（LIB-T-07，P2）不影响选区（仍指向同一章 id）。

## 8. AI 服务设计（v3.1）

### 8.1 章级分析（AI-10）

- **输入**：`chapters.content`（整章）+ 书的 `category`。
- **Prompt**：在 v3.0 `buildModernPrompt` 基础上改造为「整章」版；`category='modern'` 时跳过 `modern` 字段（prompt 中改要求）。
- **输出**：`{ analysis, explanation, modern?, summary }`。
- **缓存**：`ai_cache` 以 `scope='chapter'`、`scope_id=chapter_id`、`prompt_hash` 命中。
- **落库**：`chapter_analyses` 写 active 版本（沿用 `paragraph_analyses` 的版本化规则）。

### 8.2 对话（AI-13/14/15）

- **会话 scope**：一章一会话（`ai_threads.chapter_id`）。
- **引用**：选区作为「引用块」拼进 user 消息（system 提示词要求 AI「以下引用来自本章正文 …」）。
- **上下文**：发送时携带该 thread 最近 N（建议 8）轮消息。
- **流式**：渲染端按 token 增量渲染；主进程 `fetch` SSE 转发到渲染端 IPC `on`。
- **红线**：System Prompt 复用 `RED_LINE_PROMPT`；输出端关键词兜底不变。

### 8.3 古籍 / 现代书差异

| 能力 | 古籍 classic | 现代书 modern |
|---|---|---|
| 解读 analysis | ✅ | ✅ |
| 医理 explanation | ✅ | ✅ |
| 白话 modern | ✅ | ❌ |
| 白话 Tab | 显示 | 隐藏 |

## 10. 合规与风险（增量）

| 风险 | 说明 | 应对 |
|---|---|---|
| **选区锚定失配** | 章正文编辑后，旧摘录 / 笔记 offset 失效 | 必须 diff 重锚 + `stale` 兜底，绝不静默错位 |
| **章级 AI 成本上升** | 整章 prompt 比段大 5–10× | 章级缓存 + 控制章正文长度；超大章先提示「将按节分析」（P2） |
| **对话被诱导越界** | 用户引用一段症状描述追问用药 | 红线 System Prompt 不变；输出兜底；对话历史也送红线提示 |
| **历史数据兼容** | v3.0 段级解读 / 段笔记 | 保留，UI 不展示段解读；段笔记在章视图合并展示 |
| **树过深** | 用户无限建子节点 | 软约束最大 3 级 + 视觉提示 |

## 13. 开放问题

1. **`paragraphs` 是否长期保留**：v3.1 仅用于 FTS 与历史解读。若后续 FTS 也改为基于 `chapters.content`，可下线 `paragraphs`（需评估 trigram 索引体积）。
2. **章级 AI 对超长章的处理**：> 4k 字的章是否自动拆 prompt（影响成本与连贯性）。
3. **对话历史是否跨书共享**：当前一章一会话；是否提供「全书自由问答」入口。
4. **摘录导出**：是否在 v3.1 内补「全书摘录导出 MD」。
5. **竖排 Tab 的可访问性**：屏幕阅读器对竖排 Tab 头的支持，是否补水平 Tab 备选。
